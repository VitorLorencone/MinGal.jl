Shorthand Methods

Here we will learn about some ways to write expressions, notations, and constructions of basic structures in a more compact form, assisted by MinGal.

Selecting Elements on a Multivector

When working with Multivectors you will notice the need to select a specific element, such as a blade, grade, scalar, and so on. Below I list some ways to make this selection:

The first and most direct way is to choose the element by its index in bit-order notation, simply by using bracket notation [k]. However, since this notation is difficult to understand, it is usually not used in practice. Selection in this form begins with the grade-0 blade (the scalar) always at index 0. Thus, it is a viable option if you only want the scalar value of a multivector, but it is also worth noting that the function scalar does exactly that.

julia> Algebra(3)

julia> a = -1 + e1 + 2*e2 + 3*e3 + 4*e1e2 + 5*e1e3 + 6*e2e3 + 7*e1e2e3
- 1 + 1.0*e1 + 2.0*e2 + 4.0*e1e2 + 3.0*e3 + 5.0*e1e3 + 6.0*e2e3 + 7.0*e1e2e3 # Bit-Order

julia> a[0] # Scalar
-1

julia> scalar(a)
-1

julia> a[1] # e1
1

julia> a[2] # e2
2

julia> a[3] # e1e2
4

Another way to do the same thing is to apply the Multivector element inside the brackets, making it easier to understand and to select the scalars of each constituent blade.

julia> Algebra(3)

julia> a = -1 + e1 + 2*e2 + 3*e3 + 4*e1e2 + 5*e1e3 + 6*e2e3 + 7*e1e2e3

julia> a[id] # Scalar
-1

julia> a[e1]
1

julia> a[e2e3]
6

Finally, grade selection has a dedicated function for this, grade_selection, which selects all blades of a given grade.

julia> Algebra(3)

julia> a = -1 + e1 + 2*e2 + 3*e3 + 4*e1e2 + 5*e1e3 + 6*e2e3 + 7*e1e2e3

julia> grade_selection(a, 0)
- 1

julia> grade_selection(a, 1)
1.0*e1 + 2.0*e2 + 3.0*e3

julia> grade_selection(a, 2)
4.0*e1e2 + 5.0*e1e3 + 6.0*e2e3

julia> grade_selection(a, 3)
7.0*e1e2e3

Setting Elements on a Multivector

As described in the section above, the same constructions can be used, as can be seen in the example below:

julia> Algebra(3)

julia> a = -1 + e1 + 2*e2 + 3*e3 + 4*e1e2 + 5*e1e3 + 6*e2e3 + 7*e1e2e3

julia> a[0] = 3
3 + 1.0*e1 + 2.0*e2 + 4.0*e1e2 + 3.0*e3 + 5.0*e1e3 + 6.0*e2e3 + 7.0*e1e2e3

julia> a[e1e2] = 0
3 + 1.0*e1 + 2.0*e2 + 3.0*e3 + 5.0*e1e3 + 6.0*e2e3 + 7.0*e1e2e3

Key and Values

In order to verify the existence of elements and their values in a Multivector, we can use the special functions below. Furthermore, they provide possibilities for more controlled iteration when needed.

The keys function returns the indices in bit-order of all elements present in the multivector’s internal structure. The values function does the same, in the same order, but with the values associated with each of those indices, that is, the scalar value of each key.

julia> Algebra(3)

julia> a = -1 + e1 + 2*e2 + e2e3 + 7*e1e2e3

julia> keys(a)
[0, 1, 2, 6, 7]

julia> values(a)
[-1, 1, 2, 1, 7]

julia> res = 0
julia> for (i, val) in zip(keys(a), values(a))
           res += Blade(i, val)
       end

julia> res
- 1 + 1.0*e1 + 2.0*e2 + 1.0*e2e3 + 7.0*e1e2e3

Finally, we can check whether a key exists using the haskey function. It works both with the numeric index and with the written representation of the corresponding multivector.

julia> Algebra(3)

julia> a = -1 + e1 + 2*e2 + e2e3 + 7*e1e2e3

julia> haskey(a, 1) # e1
true

julia> haskey(a, e1e3)
false

julia> haskey(a, eI)
true

Size and Length

Two useful functions allow you to determine the number of elements in a multivector and also its maximum possible size. The size function is more internal and always returns the maximum value for the algebra of that multivector. The length function, on the other hand, returns the number of elements currently present in the multivector’s internal structure at the moment it is called.

julia> Algebra(3)

julia> a = -1 + e1 + 2*e2 + e2e3 + 7*e1e2e3

julia> size(a)
8

julia> length(a)
5

julia> a += 1 - e1

julia> length(a)
3

Iterating over a Multivector

In short, iterating over multivectors is quite simple and can be done in several ways, as demonstrated above. However, in a more simplified manner, we can do:

julia> Algebra(3)

julia> a = -1 + e1 + 2*e2 + e2e3 + 7*e1e2e3

julia> for blade in a
           println(blade)
       end
- 1
1.0*e1
2.0*e2
1.0*e2e3
7.0*e1e2e3

If you want to iterate only over the canonical basis blades of an algebra, you can use the function canonical_basis, thus it is also possible to write:

julia> al = Algebra(3)

julia> for blade in canonical_basis(al)
           println(blade)
       end
1.0*e1
1.0*e2
1.0*e3

julia> for (index, blade) in pairs(canonical_basis())
           println(index*blade)
       end
1.0*e1
2.0*e2
3.0*e3

Furthermore, if you need greater control over the entire set of basis blades of an algebra, you can, in a slightly more complex manner, use bit manipulation by shifting the bit-order index to return the element you want. You can also simply proceed as you normally would, for example:

julia> al = Algebra(3)
julia> n = al.p + al.q + al.r

julia> for i in 0:(al.max-1)
           println(Blade(i,1))
       end
1
1.0*e1
1.0*e2
1.0*e1e2
1.0*e3
1.0*e1e3
1.0*e2e3
1.0*e1e2e3

julia> for j in 1:n
           println(Blade(1<<(j-1),1))
       end
1.0*e1
1.0*e2
1.0*e3

If it is necessary to iterate over a GAVector or similar structures in order to combine all elements into a single operation, a simpler alternative to using a basic for loop is to apply Julia’s reduce method to simplify the process, which is very useful in various geometric algebra computations.

julia> Algebra(3)

julia> vec = [e1, e3, e2, id] 

julia> typeof(vec) <: GAVector
true

julia> reduce(^, vec; init=Blade(1)) # same as e1^e3^e2^id
- 1.0*e1e2e3

Finally, the chain method is also useful for helping associate a vector of elements with the canonical basis vectors, which is very handy for converting matrix rows into geometric-algebra multivectors.

julia> Algebra(3)

julia> vec = [1, 2, 3]

julia> chain(vec)
1.0*e1 + 2.0*e2 + 3.0*e3
